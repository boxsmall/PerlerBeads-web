# 拼豆图生成网站 - 实现计划

## 背景

用户想要创建一个网站，可以上传图片并将其转换为拼豆（Perler Beads）图案，支持选择输出图案的尺寸。

## 项目阶段

### 第一阶段：项目初始化
- 初始化一个 Web 项目（考虑使用纯 HTML/JS 或轻量框架）
- 项目结构规划

### 第二阶段：核心功能实现

#### 2.1 图片上传功能
- 支持拖拽上传和点击选择
- 支持常见图片格式（PNG、JPG、GIF 等）
- 图片预览功能
- **新增**：上传后支持拖动图片位置（平移/缩放）以调整构图

#### 2.2 图像处理算法
- 将图像像素映射到拼豆颜色
- 实现颜色量化（使用 K-means 或中位切分算法）
- 降噪和边缘处理
- 保持细节同时减少颜色数量
- **新增**：支持两种模式 - 像素模式（清晰识别每个像素）和轮廓模式（识别轮廓边缘）

#### 2.3 尺寸选择
- 提供预设尺寸选项（如 11x11, 20x20, 29x29 等）
- 支持自定义尺寸输入（范围限制 5x5 - 50x50）
- **新增**：自定义宽高（不一定是正方形）

#### 2.4 图案生成与展示
- 将处理后的图案渲染到画布
- 显示颜色对照图/图例
- 缩放和平移查看细节

#### 2.5 导出功能
- 导出为图片格式（PNG）
- 导出为 PDF（包含颜色图例，打印用）

#### 2.6 像素编辑功能（核心）
- 点击图案中的单个像素/珠子
- 弹出颜色选择器选择新颜色
- **新增**：支持框选区域批量修改颜色
- 撤销/重做功能

#### 2.7 高级功能
- 项目保存（localStorage 或 IndexedDB）
- 设计历史记录
- 颜色调整（手动替换某些颜色）
- 预览模式（模拟实际拼豆效果）

## 技术选型

- 前端：React + Vite
- 图像处理：Canvas API
- 状态管理：React Hooks / Context
- 样式：CSS Modules 或 Styled Components

## 验证方式

1. 使用 `npm run dev` 启动开发服务器
2. 测试图片上传功能（拖拽和点击）
3. 验证不同尺寸输出的正确性（11x11, 20x20, 29x29 等）
4. 测试颜色映射是否合理
5. 测试像素编辑功能（单点修改、批量修改、撤销重做）
6. 导出 PNG 和 PDF 功能
7. 测试项目保存和历史记录功能
